理解 RxSwift 之 Observable 与 Observer

发表于 2018-12-15

前段时间入坑OC和Swift混编之后，也稍微对Rx做了一些了解。（其实是小组被迫分享…

在学习的时候发现一些概念难以理解，所以打算写篇博客记录一下。

不用多说，Rx 的核心就是 Observable，所以在看文档中肯定不断的遇到，Observable 和 Observer 两个单词。这两个单词也很好的表达出了意思，Observable（可被观察的）/ Observer(观察者)

举个🌰：
比如女朋友大喊一声饿了，你就得去买吃的。这样大概就是一个不太准确的观察者模式，女朋友就是被观察者，而你就是观察者。只要被观察的对象发出一个事件『饿了』，你（观察者）就得做出相应的响应『搞吃的去』。但是男女朋友的关系是怎么建立起来的呢？那就需要 subscribe（订阅）了。

所以梳理一下流程：观察者 -> 订阅 -> 可被观察者 -> 可被观察者发出事件 -> 观察者做出相应的响应。
